[What is branching in git and why should you use it]

A branch is an independent line of development. It lets teammates build features and fixes in parallel without breaking the main project. Branching acts as an extra safety measure for you and your teammates.

[Different Git commands and when to use them]

  • Create and switch to a new branch with git checkout -b feature-branch. In our case, we first created a development branch from main. Then, for each separate page, we created feature branches. For example, this page was made on the branch git checkout -b development-branching.

  • If we want to switch back to our development branch we can use git checkout development This allows me to move from the development-branching to development branch.

  • If we want to delete a branch, then we can use git branch -d development-branching This deletes the local branch called development-branching.
