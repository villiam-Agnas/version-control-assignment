Why are remote repositories necessary?

Remote repositories are the backbone for working together as a team on a project. They provide a central hub for teams to work on the same project by moving the project from your local device to a shared cloud. This remote repositorie on the cloud is then used to share the code by making it awailable on more local devices, it also ensures that there is a backup of the code making shure no work is lost due to your computer or device failing. Remote repositories on GitHub also tracks all work being made on the code wich is essential for a developer to be able to show their work, aswell as making everything clear on who works at what. Remmote repos also make shure that you can always go back to previous iteration of the code if something where to break wich helps with making shure you dont have a working product.
